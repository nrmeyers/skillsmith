## Task

Implement a Stripe webhook receiver at the `POST /webhooks/stripe`
endpoint in `/workspace/app/main.py`. The route must verify the
HMAC-SHA256 signature carried in the `Stripe-Signature` request
header against the raw request body, using the shared secret
available at `os.environ["STRIPE_WEBHOOK_SECRET"]`. On a
valid signature, return HTTP 200 with the JSON body
`{"received": true}`. On any signature failure (invalid HMAC,
malformed header, stale timestamp), return HTTP 400 — or 401 for
missing/malformed header per the activated fragments' guardrail.

The seed file `app/main.py` currently contains only
`from fastapi import FastAPI` and `app = FastAPI()`. Add the
webhook route to that file. Add any imports the route needs
to the top of the same file. Do not create new modules; the
task is intentionally scoped to a single file.

## Do

- Add a `POST /webhooks/stripe` route to `/workspace/app/main.py`.
- Read the raw request body as bytes (not JSON-parsed) before
  invoking signature verification.
- Verify the signature via the procedure documented in the
  skill fragments.
- Return 200 with `{"received": true}` on valid signature.
- Return 400 on invalid signature or any verification failure.

## Do not

- Do not add replay protection, idempotency tables, or any
  deduplication logic. This task is the signature-verification
  path only; replay protection is a follow-up task.
- Do not add a dead-letter queue, retry logic, or sender-side
  delivery code.
- Do not modify any file other than `app/main.py`.
- Do not import any third-party library that is not already
  listed in the pinned `pyproject.toml` (`fastapi`, `stripe`,
  `uvicorn`, `asyncpg`, `pyjwt`).
- Do not return any HTTP status other than 200 (success) or
  400/401 (rejection) from this route. Use 400 for cryptographic
  or parse-level signature failure; 401 is acceptable for missing-
  or malformed-header rejection per the webhook-patterns guardrail.

## Mechanical success criteria

- A POST to `/webhooks/stripe` with a body whose `Stripe-Signature`
  header carries a valid HMAC over that body returns HTTP 200
  with a JSON body containing `received: true`.
- A POST with the same body but a tampered (cryptographically
  invalid) `Stripe-Signature` header returns HTTP 400.
- A POST with no `Stripe-Signature` header returns HTTP 400 or 401.
- The FastAPI app starts cleanly under uvicorn after the
  modification.
